githubについて

github・・・gitという技術を利用したサービス
git・・・ソースコードの管理をするツール

（１つのアプリを複数人で開発の際）
「どこを、誰が、どのように変更したかがこんがらがるのを防ぐ」

用語
・ローカルレポジトリ・・・自分のpc内で管理するリポジトリ（pc上、ネット上ではない）
リポジトリ（倉庫）

自分のpc内で保管しているデータ（ファイルやフォルダの保管場所）

・リモートレポジトリ・・・ネットワーク上にあるリポジトリ
githubを利用し、ネット上にファイルをアップロードした状態で管理
クラウドみたいなものかな。

リモートレポジトリを同期することで、データの共有が出来る。


・クローン・・・リモートレポジトリをコピーしてローカルレポジトリを作成
（変更は加えられない）（動作確認とかコードビューが主な目的らしい）


https://qiita.com/sky0621/items/8b6e88f4327b42ade5d7
・フォーク・・・他の人のレポジトリをコピーして、github上で自分のアカウントのリポジトリとしてコピー
（変更できる）（アイデアのたたき台とか、改造などに使えるらしい）


・ブランチ・・・現行のバージョンから枝分かれさせて他の作業（機能Ａを追加、機能Ｂを追加など）
（木の枝、支流）する時に使われる機能。

枝分かれする前の「安定版」をマスターブランチ（本体）という。
本体を保持したまま、作業別（目的別）で機能の追加などが出来て、不具合があってもブランチ別
になる為対処しやすくなる。

・コミット、プッシュ・・・コミット：ローカルレポジトリに変更を反映する事
　　　　　　　　プッシュ：リモートレポジトリに変更を反映する事
コミットはゲームでいうとセーブデータが手元にある場合
プッシュはゲームでいうとセーブデータがサーバにある場合
と思う。



・プルリクエスト・・・ローカルレポジトリの変更を他の開発者に通知する機能

コードのチェックに繋がり、リリース前にコードが間違ってるまま出してしまうのを防ぐ
コードの変更箇所の表示、修正や機能追加などの行った事の「～したよ」の共有
